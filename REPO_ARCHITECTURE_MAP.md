# Repo Architecture Map

## Workspace layout

- Root `package.json` orchestrates `pnpm -r --if-present build`, delegates `test` to `artifacts/ghostlight-bot`, and exposes many verifier aliases.
- `artifacts/ghostlight-bot` is the active Discord companion runtime.
- `artifacts/api-server` is a separate built API server whose `dist/` bundle is generated by its build script.
- `lib/api-spec`, `lib/api-zod`, `lib/api-client-react`, and `lib/db` are generated/support packages.
- `scripts/` contains root verification scripts, Norwegian/trust-layer checks, and workspace support.
- `docs/`, root audit docs, and `archive/planning/` contain historical audit/planning evidence and should not be treated as active runtime code without import/script evidence.

## Ghostlight runtime map

| Area | Main paths | Notes |
|---|---|---|
| Entrypoint/bootstrap | `artifacts/ghostlight-bot/src/index.js`, `src/bot/*` | Creates Discord client, registers handlers/commands, starts scheduler registry and HTTP server. |
| Discord inbound/outbound | `src/bot/events/messageCreate.js`, `src/discord/discordSendGateway.js` | MessageCreate remains primary user-facing send path; canonical gateway is used by autonomous/system payloads. |
| Chat pipeline | `src/chat/createChatPipeline.js`, `src/chat/pipeline/*`, `src/companion/*` | Assembles model requests, prompt sections, memory, safe-mode/fallback handling, adult model routing. |
| Output integrity | `src/chat/outputCorruptionDetector.js`, conversational compression tests | Blocks internal tokens, malformed JSON/SQL/code fragments, fake tool/tool-name leaks. |
| Adult routing | `src/chat/adultMode*.js`, `src/continuity/toneModeResolver.js`, `messageCreate.js !adult` | Private-channel constrained; normal messages in adult channel use normal model unless explicit adult intent. |
| Media/image/audio | `src/tools/mediaTools.js`, `src/images/*`, `src/bot/events/messageCreate.js`, `src/audio/*`, `src/media/*` | Direct image requests can route before LLM; LLM markers are consumed; actual Discord attachments are tested. |
| Dashboard/routes | `src/http/createHealthServer.js`, `src/http/callRoutes.js`, `src/http/renderAdminPages*`, `src/http/secondLifeApi.js` | Call route has disabled page; admin dashboard cards/routes covered by tests. |
| Automations/schedules | `src/automations/*`, `src/heartbeat/*`, `src/schedulerRegistry.js` | Background timers exist in controlled registry/automation/heartbeat layers. |
| Inner-life/autonomy | `src/innerLife/*` | Prelude, dreams, journal, self-check, autonomy posting gate and dispatch. |
| Alive/life runtime | `src/alive/*`, `src/lifeRuntime/*`, `src/life/index.js` | Alive engine and large modular life runtime expose safe status metadata and prelude contexts. |
| Memory/Qdrant | `src/memory/*`, `src/storage/*`, `src/music/library.js` | DB-backed/fallback memory, Qdrant sync, embeddings, diagnostics, memory dashboard. |
| Promise/continuity | `src/continuity/*`, pending/action files in chat/automations | Continuity selector/prelude, emotional beats, promise ledger verifiers. |
| Second Life | `src/http/secondLifeApi.js`, `src/channels/secondLifeAdapter.js`, `src/secondLife/*`, `src/companion/secondLifeReplyGenerator.js` | HTTP bridge routes and SL-specific engine modules. |
| Tests/verifiers | `src/**/*.test.js`, `test/*.test.js`, `scripts/verify-*`, `scripts/audit-*` | Large node:test suite plus many script-level verifiers. |

## Architecture boundaries observed

- Many life runtime modules have explicit tests proving they do not create schedulers or direct Discord sends; scheduling is intentionally centralized through life runtime scheduler/registry or explicit automation systems.
- Dashboard rendering is server-side string-rendered with route tests and asset manifest checks.
- Media routing is intentionally split: media tool/provider normalization, Discord message send handling, gallery persistence, dashboard/call audio paths.
