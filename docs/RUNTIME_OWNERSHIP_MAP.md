# Dante Runtime Ownership Map

Runtime Cohesion 1.0 defines ownership and boundaries for existing runtimes. This map is descriptive: it does not create new capabilities, schedulers, Discord senders, Perception Runtime, Dream Runtime, or Cognitive Runtime.

| Runtime | Owns | Reads | Writes | Emits | Consumes | Storage tables / stores | Status output | Scheduler responsibility | Sender responsibility | Degraded / unavailable behavior |
|---|---|---|---|---|---|---|---|---|---|---|
| Alive | presence, current availability signals, private activity hints | config, user presence/activity hints | alive presence store only | `prelude_refreshed` through Life Runtime when reflected in context | none directly | `alivePresenceStore` | safe presence metadata via Life status/snapshot | none; ticked/read by Life Runtime | none | marked degraded when store is not wired; snapshot uses `null` alive state |
| Inner Life | private reflections/autonomy/diagnostic channel routing | config, dispatch channel ids | existing inner-life observation surfaces | `journal_entry_created` when bridged by existing stores | diagnostic/self-check signals | existing innerLife stores/dispatch | diagnostics health only | existing inner-life scheduler if configured; not Runtime Cohesion | existing sender only; no new sender | degraded when not wired into diagnostics |
| Continuity | emotional continuity, recent decisions, prelude context | memory/context stores | existing continuity stores | `prelude_refreshed` when Life prelude updates | relationship/inner-life/context signals | continuity stores already present in codebase | safe prelude metadata | existing continuity wiring only | none | degraded when not reported; no fabricated continuity |
| Life Runtime | orchestration order, cached prelude, safe status, snapshot exposure | all runtime contexts | no domain store except existing stores through owned runtimes | `prelude_refreshed`; forwards runtime events to event bus | all runtime statuses | no new storage; uses runtime event store for cohesion events | `/api/ghostlight/life/status`, `getStatus()`, `getMindStateSnapshot()` | owns the single life runtime tick; no duplicate scheduler | none | returns skipped/degraded states instead of throwing |
| Growth | hobbies, projects, interests, skills, collections | daily plan, consequence suppression, curiosity | hobby/project/interest/skill/collection stores | `project_progressed`, `project_completed`, `project_abandoned` | curiosity bridge, identity bridge | `hobbyEngine`, `projectEngine`, `interestDriftEngine`, `skillGrowthEngine`, `collectionsEngine` stores | safe growth context | ticked by Life Runtime only | none | degraded if engines missing; growth context is `null` |
| Curiosity | private questions, attention focus, insights | daily plan, growth context, consequences | private question, attention, insight stores | `curiosity_matured`, `insight_created` | growth context, repair/give-space state | `privateQuestionStore`, `attentionDriftEngine`, `insightEngine` stores | safe curiosity context | ticked by Life Runtime only | none | degraded if engines missing; open/mature counts default safe |
| Relationship State | canonical relationship read model | relationship weather/context and consequences | in-memory snapshot only | `relationship_weather_changed`, `repair_started`, `repair_completed` | consequence context | none beyond relationship stores | `relationshipState` safe metadata | built by Life Runtime after relationship/consequence refresh | none | degraded if weather engine missing; snapshot reports source health |
| Relational Consequences | active consequences, suppression, repair carryover | interaction signals, repair analysis, active consequence store | consequence store and existing micro-life reflection events | `consequence_created`, `repair_started`, `repair_completed` | relationship state, chat observe hook | `consequenceStore` | safe counts/severity/repair flags | reviewed by Life Runtime only | none | disabled/missing means no consequence creation; actions are not falsely suppressed |
| Homeostasis | needs, need drift/momentum, first experiences, reactive fulfillment planning | daily plan, alive presence, relationship/growth/curiosity contexts | needs store, momentum, fulfillment logs, first-experience queue | `need_changed`, `need_satisfied`, `need_depleted`, `first_experience_recorded` | consequence gates, fulfillment planner/executor | `needsStore`, `fulfillmentLogStore`, `needMomentumEngine`, `firstExperienceStore` | safe need counts/top plan metadata | ticked by Life Runtime only | none; executor may use existing adapters but not Discord sender | missing stores fall back to defaults/in-memory context; no fake fulfillment |
| Identity | values, beliefs, preferences, boundaries, identity journal | homeostasis and consequence context | identity value/belief/preference/boundary/journal stores | `identity_value_changed`, `identity_belief_changed`, `identity_preference_changed`, `journal_entry_created` | first-experience queue, repair/restraint signals | identity stores | safe identity context/status | ticked by Life Runtime only | none | degraded if runtime missing; no replacement personality |
| Fulfillment | proactive agency planning, adapter execution, evidence-backed outcomes | homeostasis, identity, capability context | fulfillment history and resource library | `fulfillment_succeeded`, `fulfillment_failed`, `fulfillment_deferred`, `resource_discovered` | identity guidance, consequence gates | `fulfillmentHistoryStore`, `resourceLibraryStore` | safe outcome/adapter metadata | ticked by Life Runtime only | no Discord sender; uses existing action adapters | no fabricated success; missing executor/store degrades to no outcome |
| Diagnostics | self-check status, diagnostic carry-forward metadata | self-consistency monitor, inner-life wiring, config | last self-check in memory | `diagnostic_warning`, `self_confidence_low` | self-consistency signals | no new storage | diagnostics object in Life status/snapshot | existing diagnostic/self-check mechanisms only | existing diagnostic channels only; no new sender | reports missing monitor/innerLife as degraded/not wired |
| Self Consistency | reply self-trust, duplicate/tone/repair risk signals | recent interaction, fulfillment evidence, memory context, relationship state | in-memory last signal/prelude warning | `self_confidence_low`, `diagnostic_warning` | observeInteraction hook | none | selfConsistency status and diagnostics | no scheduler | none | degraded if monitor missing; warning omitted rather than fabricated |

## Canonical event spine

Runtime Cohesion uses `runtimeEventBus` and `runtimeEventStore` as a lightweight internal event spine. The bus has no scheduler and no Discord sender. It stores safe, stripped events and falls back to memory when the store is unavailable.

Event shape:

```js
{
  id, companion_id, customer_id, event_type, source_runtime, target_runtime,
  summary, evidence_ids, payload, confidence, created_at
}
```
